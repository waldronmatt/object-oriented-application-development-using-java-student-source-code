If you have difficulties compiling the programs you write for Chapter 17, you may need to compile them using the JDK, rather than with software you normally use.  To compile using the JDK:

                          

1.         Open an MS-DOS window.

 

2.         At the C:\windows prompt type

      cd\jswdk-1.0.1\classes\package_name

 

where package_name is the package name (if any) for your class files. (This assumes that C:\jswdk-1.0.1 is the installation directory for the JSWDK.)

 

3.         Type dir and press Enter. You will see a list of the files that are in this folder.  Make sure that the file you want to compile is contained in the list. 

 

4.         At the C:\jswdk-1.0.1\classes\package_name prompt type

 

                        set classpath=c:\jswdk-1.0.1\classes

 

This tells the java compiler to look in the classes folder (and its subfolders) for other class files that are needed.

 

5.         Type  javac filename.java  where filename.java is the name of the program you want tocompile.

 

6.         If there are no compiler errors, the compiler will produce a filename.class file.  If this is a servlet, copy the class file to the appropriate folder in the jswdk-1.0.1\webpages\WEB-INF\servlets folder.

 

 

If you have difficulties running the programs you write for Chapter 17:

 

1.         Make sure that you have registered the Data Source Name for any Access databases you are using in your application.

 

2.         Make sure that the Web server is running.

 

3.         Make sure that you copy servlet class files to the appropriate folder.

 

4.         If you have modified any of the class files in your application, you must stop the server, then restart it before you will see the effect of your changes.

 

5.         Make sure that you do not have more than one copy of any class file needed by your application in the system file path.
